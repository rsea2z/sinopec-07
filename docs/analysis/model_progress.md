# 模型进展记录

## 当前结论

本项目已经完成从统一基线、模型对比、深度学习尝试到竞赛式集成与专项优化的多轮实验。整体结论不是“复杂模型全面优于简单模型”，而是期货价格序列具有很强的持久性，`naive persistence` 是一个极强且必须正视的基线。后续一切优化都应围绕“在少数确有信号的任务上谨慎地超过 naive”展开。

## 已完成实验链路

首先建立了全赛道统一工程管线，包含主力合约抽取、多频基本面对齐、价格衍生特征构造以及 `T+5 / T+10 / T+20` 的直接预测。随后构建了模型对比实验，对 `naive_persistence`、`ridge_direct`、`random_forest_direct` 和 `random_forest_return` 进行了统一评估。结果表明，大多数品种与期限上，`naive persistence` 依然优于复杂模型。

在此基础上，又补充了轻量深度学习实验，包括 `MLP`、`LSTM` 和 `TransformerEncoder` 三种价格序列模型。这一轮的结果说明，深度学习模型能够学习到一部分时序结构，但在当前仅使用价格派生序列特征的设定下，尚不足以系统性超过 naive。

之后进一步采用竞赛式方法进行了验证集模型选择与加权融合搜索。该策略在 `PX` 等少数任务上出现了明确正收益，说明“复杂模型作为残差或辅助信号”是可行方向，但不能对所有品种一刀切使用。

最后又针对 `PX` 做了专项建模，将 `naive`、`random_forest_direct`、`random_forest_return` 与残差型 `ExtraTrees` 进行组合搜索，形成当前最有效的专项优化结果。

## 各轮实验的主要发现

对于原油链和塑料链，大多数任务中直接使用当前收盘价作为未来预测值已经非常强，复杂模型往往只能在个别中等预测期限上提供极小幅度的改进。对于聚酯产业链，不同品种表现差异显著。`PTA` 与 `PF` 在验证阶段容易出现“看似改进、测试退化”的现象，而 `PX` 在多轮实验中持续表现出更强的可预测性，是当前最值得深入优化的品种。

深度学习方面，`LSTM` 和 `Transformer` 通常优于 `MLP`，但仍普遍落后于 naive。这说明时序编码本身是有意义的，不过单独依赖价格序列并不足以形成显著优势。如果后续继续发展深度学习路线，应重点转向“价格序列编码 + 基本面因子融合”的混合结构，而不是继续纯价格端堆叠模型。

竞赛式集成方面，简单地在所有任务上推广复杂模型会造成明显的总成绩退化；但如果采用保守策略，只在少数经多轮实验反复显示有效的任务上启用专项模型，则可以获得稳定、可解释的提升。

## 当前最稳的策略

从目前的实验结果来看，最稳妥的策略并不是构造一个覆盖全部品种的统一强模型，而是采用分任务策略：绝大多数任务默认回退到 `naive persistence`，仅在 `PX` 上启用专项组合模型。该策略在不扩大整体风险的前提下，为 `PX` 的三个预测期限都带来了正向收益。

具体而言，`PX @ T+5` 的 RMSE 改进约为 `+2.93`，`PX @ T+10` 的 RMSE 改进约为 `+30.16`，`PX @ T+20` 的 RMSE 改进约为 `+80.98`。如果将这个保守策略视作当前版本的“竞赛提交逻辑”，那么它比“全品种统一复杂模型”更符合实际得分导向，也更符合本科论文中“稳健优先”的研究叙事。

## PX 的 attention 融合实验

在 `PX` 已经表现出稳定可学信号之后，又进一步尝试了“价格序列编码 + 精选基本面因子融合”的 attention 路线。该方案使用价格衍生时序特征构成序列输入，同时用相关性筛选得到一组静态基本面因子，并通过门控融合结构将两部分信息组合起来。

这一实验表明，attention 融合并非无效。对于中长期限，它确实能够超过 naive 基线。具体来看，`PX @ T+10` 的 RMSE 改进约为 `+37.33`，`PX @ T+20` 的 RMSE 改进约为 `+67.70`。不过，`PX @ T+5` 明显退化，说明短期价格仍然主要由强持久性主导，复杂融合反而会引入噪声。

更重要的是，虽然 attention 融合对 `PX` 的中长期限有效，但它目前仍未超过现有的 `PX specialist`。因此在当前版本中，attention 融合更适合作为论文中“进一步改进尝试且验证有效的结构化深度学习方案”，而不是最终的最优提交模型。

## 下一步建议

下一阶段建议沿两条路线并行推进。第一条路线是继续打磨 `PX`，但重点不再是“是否引入 attention”，而是“如何让 attention 融合超过当前 specialist”，例如改进静态因子筛选、按期限单独配置、或做更保守的融合门控。第二条路线是将当前工程与分析文档继续整理成更完整的毕业论文支撑材料，包括实验设计说明、表格化结果总结以及后续高质量图表输出。
